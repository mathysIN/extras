# User Preferences

## Communication Style
- Be concise and direct
- No unnecessary explanations
- One-word answers when possible
- No emojis
- Avoid preambles and postambles

## Workflow
- Prefer simple solutions over complex ones
- Ask before creating new files
- Use existing code patterns
- Keep scripts minimal

## Sudo/Permissions
- Use `sudo -E` to preserve environment
- Prefer ACLs or groups over passwordless sudo when possible
- Ask before modifying system files

## Code Style
- Minimal comments
- No boilerplate
- Function over form
